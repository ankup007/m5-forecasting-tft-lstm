# M5 Competition - Script Inventory

This document maps the scripts in the `scripts/` directory to their roles in the current architecture.

## 🚀 Core Modeling & Execution
These scripts are the main entry points for running experiments. They typically wrap internal library logic from `src/deepar_m5/`.

- **`train_deepar_m5.py`**: Entry point for training a single DeepAR model.
- **`predict_deepar_m5.py`**: Entry point for generating predictions from a trained checkpoint.
- **`run_deepar_m5_experiments.py`**: Entry point for running multiple experiments (sweeps/grid search).
- **`evaluate_deepar_m5.py`**: Post-hoc evaluation of a checkpoint on validation/test sets.
- **`smoke_deepar_m5.py`**: Lightweight "smoke test" to verify the training pipeline on a tiny subset of data.

## 📊 Dashboard & Visualization Pipeline
These scripts build the interactive HTML dashboard located in `docs/`.

- **`build_experiment_html_viewer.py`**: **[PRIMARY]** Generates the main `index.html` dashboard. It scans the `docs/` directory for experiment runs and aggregates them into a single-page app.
- **`calculate_run_wrmsse.py`**: **[NEW]** Calculates hierarchical WRMSSE for a specific run and saves `wrmsse.json`. This is required for competitive score display in the dashboard.
- **`build_baseline_summary.py`**: Processes naive/seasonal-naive forecasts to create `baseline_summary.json`. This provides the global "baseline" comparison in the dashboard.
- **`build_series_json_artifacts.py`**: Converts raw experiment CSVs/logs into the specific JSON format required by the dashboard's interactive plots.
- **`build_github_pages_bundle.py`**: Copies a run into `docs/` and rebuilds `docs/index.html` for GitHub Pages or local viewing.

To view the dashboard locally after bundling, serve `docs/` over HTTP instead of opening `index.html` directly from disk:

```powershell
python -m http.server 8000 --directory docs
```

Then open `http://127.0.0.1:8000/index.html`.

## 🛠️ Specialized Utilities
Tools for specific metrics, verification, or legacy support.

- **`score_holdout_forecasts.py`**: Calculates standard metrics (MAE, RMSE, RMSSE) for forecast CSVs. Usually called as part of the pipeline before `build_series_json_artifacts.py`.
- **`verify_wrmsse_naive.py`**: A verification script to ensure our WRMSSE implementation matches the naive baseline expectations.
- **`calculate_run_wrmsse.py`**: Optimized version of WRMSSE calculation (handles full 30k series hierarchy).

## ⚠️ Deprecated / Legacy
These scripts were used in previous versions of the UI and are mostly kept for reference.

- **`build_run_html_viewer.py`**: Older version of the HTML builder (single-run focus).
- **`build_series_html_report.py`**: Generates static HTML reports for individual series (superseded by the interactive dashboard).
- **`experiment_dashboard.py`**: A Streamlit-based dashboard. This was the previous UI before moving to the current lightweight HTML/JS version.
- **`generate_demo_plots.py`**: Used for generating static matplotlib plots for documentation.

---
*Note: The current dashboard relies primarily on `build_experiment_html_viewer.py` and the JSON artifacts generated by `build_series_json_artifacts.py` and `calculate_run_wrmsse.py`.*
